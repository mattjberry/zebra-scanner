# Zebra Scanner

A novelty web app that reads a zebra's stripes as a barcode, decodes them into a UPC, and finds the nearest matching grocery product in the Open Food Facts database.

Upload a zebra, then watch the image processing pipeline run live and find out your zebra is a Blueberry Yogurt Parfait.

---

## How It Works

1. A YOLO model detects the zebra and crops the image to the subject
2. The cropped image is converted to grayscale, thresholded, cleaned, and aligned
3. The stripe pattern is collapsed to a 1D signal and decoded into a 12-digit UPC
4. The nearest known UPC is found via binary search on a local Open Food Facts index
5. The matched product is fetched from the Open Food Facts API

Each step streams back to the browser in real time via Server-Sent Events.

## Stack

| Layer | Technology |
|---|---|
| Frontend | React + Vite |
| Backend | Django + Django REST Framework |

- Docker and Docker Compose

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/your-username/zebra-barcode.git
cd zebra-barcode
```

### 2. Create your `.env` file

```bash
cp .env.example .env
```

Open `.env` and fill in the values:

```env
DJANGO_SECRET_KEY=your-secret-key-here
DEBUG=True
USER_AGENT=ZebraBarcode/1.0 (your-contact-here)
```

Generate a secret key with:

```bash
python3 -c "import secrets; print(secrets.token_urlsafe(50))"
```

### 3. Build the UPC index (one-time setup)

This downloads the Open Food Facts database export (~1.5GB), extracts all valid product codes, and writes a sorted index file (~40MB) that ships with the backend container. It only needs to run once but can be updated periodically to get any updates made to the database.

```bash
# Activate your Python environment first
source backend/.venv/bin/activate   # or your preferred environment

pip install requests
python3 build_upc_index.py
```

This takes a few minutes. The output is written to `backend/zebra/data/upc_index.txt`.

### 4. Local Build and Run

```bash
docker compose up --build
```

- **Frontend**  http://localhost:5173
- **Backend API** http://localhost:8000

Subsequent starts (no code changes) can skip the `--build` flag:

```bash
docker compose up
```

## Development

Both containers run with live reload via volume mounts — changes to source files on the host are reflected immediately without rebuilding.

**Backend** (Django) picks up Python file changes automatically via its file watcher.

**Frontend** (Vite) hot-reloads on save.

To run a command inside a container:

```bash
docker compose exec backend python3 -c "from zebra.lookup import lookup_upc; print(lookup_upc('3017620422003'))"
docker compose exec frontend npm run build
```

## Notes

- The app is stateless — no images or results are stored anywhere
- The YOLO model weights (~6MB) are downloaded at Docker build time
- Open Food Facts coverage is strongest for European grocery products, sometimes you get results in french lol
- Some zebra photos will degenerate to all zeros
